# 强哥备份工具 V2 企划书

---

## 一、产品概述

**产品名称**：强哥备份工具

**产品定位**：面向广联达等工程计价软件（GBQ/GSH/GSC 系列）的 Windows 桌面文件级实时备份工具。以系统托盘形式常驻后台，实时监测工程文件变更并自动创建加密备份，防止误操作、软件崩溃及反盗版扫描导致文件丢失。

**适用场景**：从事工程造价、招投标等工作的专业人员，日常频繁修改 GBQ/GSH/GSC 等工程文件，需要一份"随时可回退"的本地安全保障。

**运行环境**：Windows 10/11，64 位。

**安装方式**：绿色免安装，单一 exe 文件，直接运行即可。

---

## 二、核心设计原则

### 原则 1：备份即真相（Backup as Source of Truth）

软件不维护独立的"文件状态索引文件"。备份存档本身就是唯一的真相来源。判断一个文件是否需要备份时，软件直接拿当前文件与它的最新备份存档做对比，而不是去查某个中间状态表。

**这一原则的动机**：

旧版使用一个 `backup_state.json` 文件记录每个文件的哈希值作为"基准"。当软件关闭期间文件被外部修改（例如在 Windows 资源管理器中复制替换），这个基准就过时了——再次打开软件时，旧的基准会认为文件"没有变化"，导致漏备份。去掉这个中间状态文件之后，软件永远拿"最新一份备份存档"当参照物，参照物不会过时。

**对用户而言这意味着**：

- 不需要理解什么是"状态文件"
- 不需要执行"重建索引"（此功能在新版中不存在）
- 无论软件是否在运行，只要改过文件，下次启动软件后它都会正确识别并提醒备份

### 原则 2：静默守护

正常运行时软件不打扰用户。托盘图标常驻，鼠标悬停可看到当前状态。仅在下列情况下主动弹窗通知：

- 监控过程中一次性变更文件数超过安全阈值（防止误操作、勒索软件、磁盘异常等）
- 启动时发现需要备份的文件（告知数量但不自动备份）
- 启动时发现孤儿备份（告知数量但不自动删除）
- 备份或恢复操作完成或失败

### 原则 3：加密混淆

所有备份存档使用 AES-256-CBC 加密，并开启头部加密（即压缩包内的文件名也是加密的）。任何人（包括广联达反盗版扫描程序）在不输入密码的情况下无法看到压缩包中存放了什么文件。

重要说明：此加密的目的是**混淆**而非**防人**——防止广联达在本地扫描磁盘时识别出备份文件。密码以明文存入配置文件，熟悉的人仍可获取。如需要真正的安全防护，请使用第三方加密工具。

### 原则 4：最小交互

每个核心操作（手动备份、恢复文件、清理孤儿备份、完整性检查）在 3 次点击内完成。

## 三、核心机制详解

### 3.1 变化检测

**这是整个软件最重要的机制。** 它决定了"软件如何知道一个文件需要备份"。

#### 工作原理

每次需要判断文件是否需要备份时（启动扫描、手动备份、实时监控捕获到变更），对每个候选文件执行以下逻辑：

1. **找到该文件的最新备份存档**：根据源文件的路径，在备份目录中找到对应的子目录。在该子目录中找出所有以该文件名为前缀的存档。按存档名中的时间戳排序，取最新的一个。
2. **提取存档中的文件哈希**：备份存档的文件名中嵌入了备份时计算的文件 SHA-256 哈希的前 16 位。直接读文件名即可获取，不需要解压存档。
3. **计算当前文件的哈希**：计算当前文件的 SHA-256，取前 16 位。
4. **对比**：如果 16 位哈希不同，说明文件内容自上次备份后发生了变化，标记为"需要备份"。
5. **找不到存档**：如果该文件在备份目录中没有任何存档（新文件），直接标记为"需要备份"。

#### 可靠性分析

- 参照物永远是最新的备份存档，不会过时。
- **文件改名/移动**：旧路径的存档成为孤儿备份（后续可清理）；新路径的文件因找不到旧存档而被视为新文件，从头开始备份。不会丢失数据，但不保留改名前的备份历史连续性。
- **改名后又改回原名**：恢复到原路径后，匹配到原有备份系列，只备份变化的内容。
- **备份存档被手动删除**：该文件下次扫描时"找不到历史存档"，视为新文件直接备份。

#### 与 OneDrive 的类比

OneDrive 不维护本地状态文件——它直接比较本地文件的修改时间和大小与云端最新版本。如果软件关闭期间文件被修改，下次启动 OneDrive 后它直接发现本地文件和云端版本不一致，触发同步。本软件的逻辑完全一致：把最新备份存档当作"云端版本"，拿当前文件与之对比。

---

### 3.2 实时文件监控

在用户点击"开始监控"后启动。使用 Windows 内核级文件系统事件机制（非轮询），对源目录中匹配的文件类型进行监控。

**事件类型**：文件修改、创建、删除、重命名。

**防抖机制**：文件保存后的 3 秒内（可配置），如果同一文件再次触发事件，计时器重置。3 秒内没有新事件，才执行备份。避免了"保存一次文件触发多次备份"的问题。

**异常保护**：如果在一次防抖窗口内，变更的文件或文件夹数量超过阈值（默认 3 个），软件自动停止监控，托盘弹出警告通知。这是防止以下情况的安全措施：

- 误操作批量复制覆盖了大量文件
- 勒索软件加密了目录中的文件
- 磁盘错误导致大量文件异常变更

**文件过滤**：忽略 `.7z`、`.tmp` 文件和以 `~` 结尾的临时文件。

---

### 3.3 手动备份

不依赖实时监控，用户主动触发。

**操作流程**：

1. 点击"手动备份"
2. 软件扫描所有源目录，识别所有需要备份的文件
3. 弹窗显示：变更文件数量、总大小、预计耗时
4. 用户确认后开始备份
5. 备份完成后显示结果

**并发处理**：多源目录可并行备份（最多 4 个线程）。

---

### 3.4 加密备份存储

#### 存档格式

每个源文件对应独立的 7z 加密存档。不是把所有文件打包成一个——每个文件有自己的 7z 文件。

#### 加密参数

- 算法：AES-256-CBC
- 头部加密：开启（文件名在压缩包内不可见）
- 压缩方式：仅存储，不压缩（工程文件通常已经是压缩或二进制格式，二次压缩无意义）

#### 目录结构

备份目录结构完全镜像源目录（去掉盘符）：

```
源文件：  C:\项目\预算部\2025投标\报价总表.GBQ7
备份存档：D:\强哥备份\项目\预算部\2025投标\报价总表_20260525_143022_a1b2c3d4e5f6789.7z
                                原始文件名    日期      时间    文件哈希(16位)
```

#### 存档命名规则

```
{原始文件名}_{YYYYMMDD}_{HHMMSS}_{16位SHA-256前16位}.7z
```

存档名中的点号 `.` 和空格替换为下划线 `_`，避免路径解析问题。

#### 版本保留

每个文件最多保留 N 个版本（默认 5，可配置 1-99）。每次成功创建新备份后，自动删除该文件最旧的备份存档，使总数不超过限制。

#### 原子写入

备份写入过程：先写入 `文件名.tmp`，写入完成后重命名为 `文件名.7z`。如果在写入过程中软件崩溃或断电，残留的 `.tmp` 文件不会影响正常备份，下次启动时自动清理。

---

### 3.5 一键恢复

**操作流程**：

1. 点击"一键恢复"
2. 弹窗列出所有有备份存档的文件
3. 每个文件下方展示其所有可用版本（时间倒序），最新版本默认选中
4. 可多选文件 + 为每个文件单独选择版本
5. "全选最新"按钮一键选中所有文件的最新版本
6. 点击"恢复所选"，确认覆盖提示

**恢复行为**：将选中版本的 7z 存档直接解压到源文件所在路径，覆盖当前文件。

---

### 3.6 孤儿备份清理

**孤儿备份**是指：源文件已经不存在（被删除或改名移走），但对应的备份存档还在。

**检测时机**：每次软件启动时自动扫描。发现有孤儿后托盘弹出通知（例如"发现 3 个孤儿备份"），不强制处理。

**手动触发**：用户可随时点击"孤儿清理"按钮。

**操作界面**：

- 列出所有孤儿备份，显示：原文件相对路径、版本数量、时间范围
- 支持全选/取消全选/全部保留（关闭窗口）
- 删除前二次确认

---

### 3.7 完整性检查

验证备份存档是否可正常打开和解压。存档可能因磁盘错误、意外断电等原因损坏。

**触发方式**：用户手动点击"完整性检查"按钮。

**操作界面**：

- 检查所有备份存档
- 损坏的存档显示为红色，列出路径和时间
- 支持全选/取消全选/全部保留
- 删除前二次确认

## 四、配置项说明

| 配置项 | 说明 | 默认值 | 范围 |
|--------|------|--------|------|
| 源文件夹 | 需要监控和备份的目录列表 | 无 | 可添加多个 |
| 备份存放目录 | 备份存档的根目录 | `D:\强哥备份` | 任意本地路径 |
| 文件类型 | 需要监控和备份的文件扩展名 | 37 种工程文件类型 | 逗号分隔 |
| 加密密码 | 7z 存档加密密码 | `强哥备份` | 任意字符 |
| 防抖时间 | 文件修改后等待多长时间再备份 | 3 秒 | 1-60 秒 |
| 版本上限 | 每个文件最多保留几个备份版本 | 5 个 | 1-99 |
| 异常阈值 | 单次变更超过此数量自动停止监控 | 3 个 | 1-999 |
| 开机自启 | Windows 开机时自动启动软件 | 关闭 | 开/关 |

**配置文件格式**：JSON，与软件 exe 同目录下的 `config.json`。

**默认支持的文件类型**（37 种）：

`.GBQ7`、`.GSH7`、`.GSC7`、`.GPV7`、`.GEPC7`、`.GPB7`、`.GTJ`、`.GPB6`、`.GPB5`、`.GBG9`、`.GPE9`、`.GBQ6`、`.GBQ5`、`.GBQ4`、`.GZB4`、`.GTB4`、`.GPB9`、`.GEPC6`、`.GPV6`、`.GPV5`、`.GSC6`、`.GSC5`、`.GSH6`、`.GSH5`、`.GBQSH4`、`.GBGSH4`、`.GXMSH4`、`.GDBSH4`、`.GXMDBSH4`、`.GPC5`、`.GBQPC4`、`.GBGPC9`、`.GZBPC4`、`.GTBPC4`、`.GPBPC9`、`.GPBEC9`、`.GEC5`

## 五、操作日志

软件所有关键操作记录到 `backup.log`（JSONL 格式，每行一条 JSON）。

| 类型 | 触发场景 | 记录内容 |
|------|---------|---------|
| scan | 启动自动扫描/手动备份扫描 | 扫描文件总数、发现变更数、耗时 |
| backup | 每次成功创建备份 | 源文件路径、存档路径、文件哈希、文件大小、时间戳 |
| restore | 每次成功恢复文件 | 源文件路径、存档路径、时间戳 |
| delete | 删除孤儿或损坏备份 | 删除原因、数量、存档路径列表、时间戳 |
| error | 操作异常 | 异常类型、相关文件、错误信息、时间戳 |

日志文件丢失不会影响软件任何核心功能——变化检测依赖的是备份存档本身而非日志。日志仅用于事后排查。

## 六、边界行为与异常处理

| 场景 | 软件行为 |
|------|---------|
| 软件启动，上次退出时监控是开启的 | 自动恢复监控；扫描变更文件，托盘提示数量，不自动备份 |
| 软件启动，上次退出时监控是关闭的 | 扫描变更文件，托盘提示数量；不启动监控 |
| 备份目录所在磁盘断开 | 暂停所有备份，托盘图标变红，持续检测，磁盘恢复后自动恢复 |
| 备份目录磁盘空间不足 | 备份失败，错误记录到日志，托盘提示 |
| 源文件夹在软件运行时被删除 | 该文件夹的监控停止，备份中断，日志记录 |
| 广联达正在打开工程文件时触发备份 | 尝试备份，若文件被锁定则等待 1 秒后重试，最多重试 3 次 |
| 多个源文件夹存在同名文件 | 各源文件夹独立处理，备份到各自的镜像目录，互不干扰 |
| 配置文件中源文件夹或备份目录路径无效 | 启动时不崩溃，显示"路径无效"状态，等待用户修正 |

## 八、使用指南

### 8.1 首次设置

1. 双击运行 `QiangBackup.exe`
2. 点击"添加"按钮，选择工程文件所在文件夹（可添加多个，如 `C:\项目\预算部`、`C:\项目\结算部`）
3. 设置"备份存放目录"（建议选择与源文件不同的磁盘，如 `D:\强哥备份`）
4. 确认文件类型（默认已包含所有常见广联达工程文件扩展名，一般无需修改）
5. 如需修改密码，设置新密码后务必记住
6. 根据需求调整版本上限（建议保持默认 5 个）
7. 点击"保存配置"

### 8.2 日常使用——后台监控模式（推荐）

1. 点击"开始监控"
2. 将窗口最小化（软件自动隐藏到系统托盘）
3. 正常使用广联达软件工作
4. 每次保存文件后，软件自动在 3 秒内完成备份
5. 下班时从托盘右键菜单选择"退出"，或直接关机

### 8.3 日常使用——手动备份模式

1. 工作一段时间后，点击"手动备份"
2. 确认变更文件清单和预计耗时
3. 点击确认，等待备份完成

### 8.4 恢复文件

1. 点击"一键恢复"
2. 在列表中找到需要恢复的文件
3. 选择要恢复的版本（默认选中最新）
4. 点击"恢复所选"
5. 确认覆盖提示
6. 打开广联达软件验证已恢复的文件

### 8.5 清理冗余备份

1. 点击"孤儿清理"——勾选不需要的旧档案——删除
2. 点击"完整性检查"——检查是否有损坏存档——如有则删除

### 8.6 更换电脑或迁移数据

1. 将备份目录（如 `D:\强哥备份`）完整复制到新电脑
2. 运行 `QiangBackup.exe`
3. 设置"备份存放目录"指向复制过来的备份目录路径
4. 密码必须和原来一致
5. 添加源文件夹后即可继续使用，所有历史备份自动关联

## 九、版本历史兼容

- 旧版（V1）的备份存档命名规则与 V2 完全一致，V2 可直接识别和恢复 V1 创建的存档。
- 旧版的 `config.json` 配置文件格式与 V2 兼容，无需重新配置。
- 旧版的 `backup_state.json` 文件 V2 不再使用，可手动删除。
- 旧版的 `backup.log` 日志文件 V2 可继续追加写入。

## 十、功能边界（明确不做的）

以下功能不在 V2 范围内，保持产品简洁：

- 定时/计划备份（当前仅支持手动 + 实时监控）
- 文件差异对比查看（比较同一文件不同版本的内容变化）
- 云端存储同步
- U 盘自动检测备份
- 备份统计与报告
- 备份空间预警
- 多份备份目标的镜像
- 网络/远程备份
